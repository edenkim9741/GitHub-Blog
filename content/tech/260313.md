벨로그, 네이버 블로그, 티스토리 등 여러 블로그 서비스가 난무하는 현대...
블로그를 어디에 작성할까 고민을 하다가, 결국 Github Pages로 블로그를 구축하기로 결정했다. (역시 난 비주류야 킥킥)

Github Pages는 Github에서 제공하는 무료 호스팅 서비스로, 정적 웹사이트를 쉽게 배포할 수 있다.

## Zola
나는 gemimi의 의견을 참고하여 zola를 활용해서 블로그를 구축하기로 했다.
최근에 rust가 멋있어보여서 zola를 추천했을 때 별로 거부감을 느끼지 않았다.

mac에서 작업했기 때문에 brew로 zola를 설치했다.
```bash
brew install zola
```

설치가 완료되면, zola를 사용하여 새로운 블로그 프로젝트를 생성할 수 있다.
```bash
zola init github-blog
```
이 명령어는 `github-blog`이라는 디렉토리를 생성하고, 그 안에 기본적인 블로그 구조를 만들어준다.
생성된 디렉토리로 이동하여, content 디렉토리에 목차들을 만들고 markdown 파일로 글을 작성할 수 있다.
```bash
cd github-blog/content
# 글 작성
```


zola는 [zola theme 웹사이트](https://www.getzola.org/themes/)에서 다양한 테마를 제공한다. 나는 `book`테마를 선택해서 적용했다.
```bash
cd ../themes
git clone https://github.com/getzola/book.git
```
테마를 적용하기 위해서 `zola.toml` 파일을 열어서 아래 내용을 추가한다. (내가 활용한 zola 버전은 0.22 인데, 이전 버전에서는 `config.toml` 파일인 것 같음)
```toml
theme = "book"
```

## Cusdis
나름 블로그이기 때문에 댓글 기능도 필요하다고 생각해서 cusdis를 활용하기로 했다.

cusdis는 간단한 댓글 시스템으로 무료로 사용하는 경우에 사이트 하나에 하루 댓글 100개까지 지원한다. (100개면 충분할 거 같음)
근데 조금 아쉬운 점은 내가 하나 하나 승인을 해줘야한다는 점... 추후 다른 방법으로 대체할 수도 있을 것 같다.

cusdis를 사용하기 위해서는 먼저 cusdis 웹사이트에서 계정을 만들고, 새로운 사이트를 등록해야 한다. 사이트 등록이 완료되면, cusdis에서 제공하는 스크립트 코드를 복사해서 블로그의 템플릿 파일에 추가하면 된다.

```html
    {% if page.extra.show_comments | default(value=true) %}
        <hr style="margin-top: 3rem; margin-bottom: 2rem;">
        
        <style>
            #cusdis_thread iframe {
            overflow-y: hidden !important; 
            min-height: 350px;
            }
        </style>
        
        <div id="cusdis_thread"
            data-host="https://cusdis.com"
            data-app-id="c87a1a63-2501-49b3-a770-fe5e8cf2af29"
            data-page-id="{{ page.permalink }}"
            data-page-url="{{ page.permalink }}"
            data-page-title="{{ page.title }}">
        </div>
        <script async defer src="https://cusdis.com/js/cusdis.es.js"></script>
    {% endif %}
```

## Github Pages 배포하기
블로그를 완성한 후에는 Github Pages에 배포해야 한다. 먼저, Github에 새로운 저장소를 생성하고, 로컬에서 블로그 프로젝트를 초기화한 후, Github 저장소에 연결한다. 뭐... Github Pages를 따라해볼 사람이라면 Github repository와 local repository를 연결하는 방법은 다 알 것 같아서 건너뛰고 배포하는 부분만 설명하면 아래와 같다.

.github/workflows/deploy.yml 파일을 만들어서 아래와 같이 작성한다.
```yaml
name: Zola on GitHub Pages

on:
  push:
    branches: [ main ]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repositoryj
        uses: actions/checkout@v4
        with:
          submodules: recursive 
          
      - name: Zola Install
        uses: taiki-e/install-action@v2
        with:
          tool: zola@0.22.1
          
      - name: Zola Build
        run: zola build
        
      - name: Upload artifact for deployment
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: GitHub Pages Deployment
        id: deployment
        uses: actions/deploy-pages@v4
```

이 워크플로우는 main 브랜치에 푸시될 때마다 실행되며, zola를 설치하고 블로그를 빌드한 후, Github Pages에 배포한다.
